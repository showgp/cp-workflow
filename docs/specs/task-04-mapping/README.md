## 1. 任务概述

| 属性 | 值 |
|------|-----|
| **Task ID** | task-04-mapping |
| **任务名称** | 字段映射系统 / Field Mapping System |

### 摘要

本任务是插件核心用户交互环节：用户在 UI 面板中手动建立"表格列 → 模板图层"的映射关系。系统需提供类型感知的列/层选择器、唯一性约束校验、映射清空警告、部分映射的合法化支持。该模块完全运行在 UI iframe 内，仅消费 Task 02 传入的字段列表 (`SourceTable.fields`) 和 Task 03 传入的图层列表 (`TemplateLayer[]`)，产出 `MappingEntry[]` 供 Task 05 消费。

## 2. 功能需求细节

### 2.1 FR-08: 唯一性约束

#### 2.1.1 列维度唯一性（一个列最多映射到一个图层）

- **规则**: `columnIndex` 在 `MappingEntry[]` 中最多出现一次。同一列不能同时映射到 "Title1" 和 "Title2"。
- **UI 约束**: 源列下拉框中，已映射的列**不出现**在可选列表中（而非灰化后仍可选择）。这从根本上杜绝了重复选择同一个列的可能性。
- **边界情况 — 用户通过程序化路径绕过下拉框**: 如果由于状态异常出现同一列被映射两次，在 `validateMapping()` 中检测到后，标记整个映射状态为错误态，显示全局错误横幅 "存在重复的列映射，请检查并修正"。错误行需高亮标识。

#### 2.1.2 图层维度唯一性（一个图层最多接收一个列）

- **规则**: `targetLayerId` 在 `MappingEntry[]` 中最多出现一次。不能将 "标题" 和 "副标题" 两列同时映射到同一个 "Title" 图层。
- **UI 约束**: 目标图层下拉框中，已映射的图层**不出现**在可选列表中。
- **异常检测**: 与列维度唯一性相同，若状态异常出现重复图层映射，通过验证函数捕获并展示全局错误。

#### 2.1.3 全未映射是合法状态

- 0 个列被映射、0 个图层被映射 → 完全合法。该状态下用户点击生成仅创建模板副本，不做内容替换。
- 生成前需弹出确认对话框（详见第 7 节）。

#### 2.1.4 部分未映射是合法状态

- 部分列未映射 → 这些列的数据不参与生成（FR-12）。
- 部分图层未映射 → 这些图层保留模板原始内容（FR-11）。

### 2.2 FR-09: 文本列 → 文本层映射

#### 2.2.1 类型约束

- **文本列来源**: CSV 文件的所有列均为文本列；Excel (.xlsx) 文件中除图片列外的所有列均为文本列。
- **文本层定义**: Figma 节点类型为 `TEXT` 的所有图层。
- **映射约束**: 文本列**仅可**映射到文本层，不可映射到图片层。

#### 2.2.2 文本映射行为说明

- 建立文本映射后，生成时将该列的每一行单元格文本内容写入目标文本图层的 `characters` 属性。
- 若某行单元格为空字符串或 null，该文本层保留模板原始文本（FR-25）。

### 2.3 FR-10: 图片列 → 图片层映射

#### 2.3.1 类型约束

- **图片列来源**: **仅** Excel (.xlsx) 文件可包含图片列。CSV 为纯文本格式，不支持嵌入图片。
- **图片列识别**: Task 02 在解析 Excel 时，识别包含内嵌图片对象的列，将其 `type` 标记为 `'image'`。
- **图片层定义**: Figma 节点类型为 `RECTANGLE` 且 fill 中包含 `ImagePaint` 的图层；或节点类型为 `ELLIPSE` 等形状且包含图片填充的图层。Task 03 负责识别并标记 `layerType: 'image'`。
- **映射约束**: 图片列**仅可**映射到图片层，不可映射到文本层。
- **数据性质**: 图片数据来自 Excel 单元格中**实际嵌入**的图片二进制对象，非 URL 链接。Task 02 负责提取并缓存这些图片数据。

#### 2.3.2 图片映射行为说明

- 建立图片映射后，生成时将该列当前行对应的嵌入图片设置到目标图片图层的 `fills` 属性中。
- 若某行嵌入图片提取失败，该图片位置不填充（保持空白或模板原图），该 Frame 其他字段正常填充（FR-24）。

### 2.4 FR-11: 允许部分图层不映射

- **语义**: 用户有意不将任何列映射到某些图层。这表示这些图层**保持模板原始内容不变**。
- **UI 体现**: 未映射的图层在图层下拉框中以正常选项呈现（可被选择加入映射），但在汇总信息中计入"未映射图层: Z 个"。
- **生成时行为**: 克隆模板后，未映射的图层不做任何修改。
- **与 Issue 的区别**: 未映射图层导致的"保留原始内容"不是 Issue，是预期的设计行为。

### 2.5 FR-12: 允许部分列不映射

- **语义**: 用户有意不将某些列映射到任何图层。这表示这些列的**数据不参与生成过程**。
- **UI 体现**: 未映射的列在源列下拉框中以正常选项呈现，在汇总信息中计入"未映射: Y 列"。
- **生成时行为**: 遍历每一行时，跳过未映射的列，不对任何图层产生影响。
- **典型场景**: 表格中有一列"备注"或"内部编号"，这些是辅助信息，不需要填入设计稿。

### 2.6 FR-13: 按列类型过滤可映射目标

#### 2.6.1 文本列的映射目标过滤

- 当用户在源列下拉框中选中一个**文本列**时，目标图层下拉框**仅展示文本层**（`layerType === 'text'`）作为可选项。
- 图片层不出现在文本列的目标图层下拉框中。

#### 2.6.2 图片列的映射目标过滤

- 当用户在源列下拉框中选中一个**图片列**时，目标图层下拉框**仅展示图片层**（`layerType === 'image'`）作为可选项。
- 文本层不出现在图片列的目标图层下拉框中。

#### 2.6.3 CSV 文件特殊情况

- CSV 文件所有列均为文本类型（`columnType: 'text'`）。
- 模板中可能存在图片层（layerType: `'image'`），但由于所有列都是文本列，文本列 → 图片层的映射不可能成立。因此：
  - 所有源列下拉框选择文本列后，目标图层下拉框仅显示文本层。
  - 图片层虽然在图层列表中，但不会出现在任何文本列的目标下拉框中。
  - 这自然意味着：如果模板**只有**图片层而没有文本层，且用户上传的是 CSV 文件，则无法建立任何映射（这是正确的行为，"添加映射"按钮应被禁用）。

#### 2.6.4 实现要点

- 每个映射行的源列下拉框和目标图层下拉框是**联动**的：先选源列（确定类型），再过滤目标图层选项。
- 如果用户先选目标图层再选源列（通过下拉框顺序），需做后置校验：选择的 (源列类型, 目标图层类型) 必须兼容，否则标记该行为错误状态。

### 2.7 FR-14: 模板变更时清除映射

#### 2.7.1 触发条件

- UI 接收到来自 Sandbox 的 `selection-changed` 消息，且新的模板 Frame ID 与当前 `MappingState.templateId` 不同。
- 或者 UI 接收到来自 Sandbox 的 `template-layers` 消息（新的图层列表），且其附带的 `templateId` 与当前不同。

#### 2.7.2 清除行为

1. 清空所有 `MappingEntry[]`（设为空数组 `[]`）。
2. 更新 `MappingState.templateId` 和 `templateName` 为新模板的值。
3. 显示警告横幅 "模板已变更，所有映射已清除。请重新建立映射。"。
4. 横幅可被用户手动关闭（点击 ×），关闭后横幅消失，但映射不会被恢复。

#### 2.7.3 数据源变更时的同等处理

- 为简化 MVP 逻辑，当 Task 02 通知字段列表变更（用户重新上传文件或选择不同文件）时，同样清除所有映射并显示警告横幅 "数据源已变更，所有映射已清除。请重新建立映射。"。
- 原因：列索引和列类型可能与之前不同，保留旧映射会造成混乱。

## 3. 核心数据模型

### 3.1 类型定义

```typescript
/**
 * 单个映射条目：一个表格列 → 一个模板图层
 */
interface MappingEntry {
  /** 唯一标识，用于 React key 和操作追踪，UUID v4 */
  id: string;

  /** 列在 SourceTable.fields 中的索引 */
  columnIndex: number;

  /** 列的名称，用于显示 */
  columnName: string;

  /** 列的数据类型 */
  columnType: 'text' | 'image';

  /** 目标图层的 Figma 节点 ID */
  targetLayerId: string;

  /** 目标图层的显示名称 */
  targetLayerName: string;

  /**
   * 目标图层的完整路径，用于同名图层区分
   * 示例: "Frame 1 > Group 3 > Title"
   */
  targetLayerPath: string;
}

/**
 * 映射系统的完整状态
 */
interface MappingState {
  /** 所有活跃的映射条目 */
  entries: MappingEntry[];

  /** 当前这些映射所属的模板 ID，用于检测模板变更 */
  templateId: string | null;

  /** 模板名称，用于展示 */
  templateName: string | null;
}

/**
 * 验证结果结构
 */
interface ValidationResult {
  /** 是否通过所有验证 */
  valid: boolean;

  /** 错误列表（若 valid 为 false） */
  errors: ValidationError[];
}

interface ValidationError {
  /** 错误类型 */
  type: 'DUPLICATE_COLUMN'
    | 'DUPLICATE_LAYER'
    | 'TYPE_MISMATCH'
    | 'INVALID_LAYER_ID'
    | 'INVALID_COLUMN_INDEX';

  /** 出错映射条目的 ID */
  entryId: string;

  /** 人类可读的错误消息 */
  message: string;
}
```

### 3.2 辅助类型（来自其他 Task，此处仅作引用说明）

映射系统消费但不定义以下类型：

```typescript
// ——— 来自 Task 01（架构）———

/** 源数据表格 */
interface SourceTable {
  fields: FieldDef[];      // 列定义
  rows: RowData[];         // 行数据
  fileName: string;
  fileType: 'csv' | 'xlsx';
}

interface FieldDef {
  index: number;           // 列索引
  name: string;            // 列名（无表头时为 "列 A" 等）

}

// ——— 来自 Task 03（模板选择）———

interface TemplateLayer {
  id: string;              // Figma 节点 ID
  name: string;            // 节点名称
  path: string;            // 完整路径
  layerType: 'text' | 'image';
  nestedFrameParentId?: string;
}
```

### 3.3 验证函数规范

#### 3.3.1 `validateMapping(entries, columnIndex, targetLayerId): ValidationResult`

**职责**: 在添加或修改一条映射前，验证列和图层是否满足约束。

**验证步骤**（按序执行，第一个失败即返回）:

1. **图层存在性校验**（可选，因图层列表来自 Task 03 已保证）: 检查 `targetLayerId` 是否在可用图层列表中。
2. **列已映射检查**: 遍历 `entries`，检查 `columnIndex` 是否已被其他 entry 占用。
   - 若 `entries` 中已存在 `e.columnIndex === columnIndex`（且 `e.id !== 当前entryId`），返回错误 `DUPLICATE_COLUMN`。
3. **图层已映射检查**: 遍历 `entries`，检查 `targetLayerId` 是否已被其他 entry 占用。
   - 若 `entries` 中已存在 `e.targetLayerId === targetLayerId`（且 `e.id !== 当前entryId`），返回错误 `DUPLICATE_LAYER`。
4. **类型兼容检查**: 根据 `columnIndex` 找到列类型（`text` / `image`），根据 `targetLayerId` 找到图层类型（`text` / `image`），二者必须一致。
   - 若不一致，返回错误 `TYPE_MISMATCH`。

**返回**:
- `{ valid: true, errors: [] }` 通过验证。
- `{ valid: false, errors: [...] }` 验证失败，errors 中包含具体的错误信息。

#### 3.3.2 `isColumnMapped(entries, columnIndex): boolean`

**职责**: 检查指定列是否已被映射。

```typescript
function isColumnMapped(
  entries: MappingEntry[],
  columnIndex: number
): boolean {
  return entries.some(e => e.columnIndex === columnIndex);
}
```

#### 3.3.3 `isLayerMapped(entries, targetLayerId): boolean`

**职责**: 检查指定图层是否已被映射。

```typescript
function isLayerMapped(
  entries: MappingEntry[],
  targetLayerId: string
): boolean {
  return entries.some(e => e.targetLayerId === targetLayerId);
}
```

#### 3.3.4 `getMappedColumnIndices(entries): Set<number>`

**职责**: 获取所有已被映射的列索引集合。

```typescript
function getMappedColumnIndices(entries: MappingEntry[]): Set<number> {
  return new Set(entries.map(e => e.columnIndex));
}
```

#### 3.3.5 `getMappedLayerIds(entries): Set<string>`

**职责**: 获取所有已被映射的图层 ID 集合。

```typescript
function getMappedLayerIds(entries: MappingEntry[]): Set<string> {
  return new Set(entries.map(e => e.targetLayerId));
}
```

#### 3.3.6 `getUnmappedColumns(fields, entries): FieldDef[]`

**职责**: 获取尚未被映射的列列表。

```typescript
function getUnmappedColumns(
  fields: FieldDef[],
  entries: MappingEntry[]
): FieldDef[] {
  const mappedIndices = getMappedColumnIndices(entries);
  return fields.filter(f => !mappedIndices.has(f.index));
}
```

#### 3.3.7 `getUnmappedLayers(layers, entries): TemplateLayer[]`

**职责**: 获取尚未被映射的图层列表。

```typescript
function getUnmappedLayers(
  layers: TemplateLayer[],
  entries: MappingEntry[]
): TemplateLayer[] {
  const mappedIds = getMappedLayerIds(entries);
  return layers.filter(l => !mappedIds.has(l.id));
}
```

#### 3.3.8 `getCompatibleLayers(layers, columnType): TemplateLayer[]`

**职责**: 获取与指定列类型兼容的图层列表。

```typescript
function getCompatibleLayers(
  layers: TemplateLayer[],
  columnType: 'text' | 'image'
): TemplateLayer[] {
  return layers.filter(l => l.layerType === columnType);
}
```

## 4. UI 组件规格

### 4.1 整体布局

映射区域为插件面板中的一个独立 section，采用垂直排列的卡片式布局。

```
┌──────────────────────────────────────────────────────────────┐
│  字段映射                                                     │
│                                                              │
│  ┌──────────────────────────────────────────────────────────┐│
│  │ 表格列                        模板图层                    ││
│  │ ┌────────────────┐  →  ┌───────────────────────┐  [×]   ││
│  │ │ 标题 (文本)     │     │ Frame1/Grp/Title (文本) │       ││
│  │ └────────────────┘     └───────────────────────┘         ││
│  └──────────────────────────────────────────────────────────┘│
│                                                              │
│  ┌──────────────────────────────────────────────────────────┐│
│  │ ┌────────────────┐  →  ┌───────────────────────┐  [×]   ││
│  │ │ 图片 (图片)     │     │ Frame1/Grp/Photo (图片) │       ││
│  │ └────────────────┘     └───────────────────────┘         ││
│  └──────────────────────────────────────────────────────────┘│
│                                                              │
│  ┌──────────────────────────────────────────────────────────┐│
│  │ ┌────────────────┐  →  ┌────────────────────┐     [×]   ││
│  │ │ 请选择…        │     │ 请选择…             │           ││
│  │ └────────────────┘     └────────────────────┘            ││
│  └──────────────────────────────────────────────────────────┘│
│                                                              │
│  [+ 添加映射]                                                │
│                                                              │
│  ─────────────────────────────────────────────────────────── │
│  已映射: 2 列  |  未映射: 3 列  |  未映射图层: 1 个            │
└──────────────────────────────────────────────────────────────┘
```

**布局约束**:
- 映射区域最大高度: 当映射行超过 5 行时，内部滚动（`overflow-y: auto`），"+ 添加映射"按钮和汇总信息栏固定在底部不随滚动。
- 最小宽度: 300px（适配 Figma 插件面板宽度）。
- 各元素间距: 8px（Figma 设计系统常用间距）。

### 4.2 映射行 (MappingRow) 的组成

每个映射行是一个水平布局的 flex 容器，包含以下子元素：

```
[源列选择器]  →  [目标图层选择器]  [×]
```

#### 4.2.1 源列选择器 (Source Column Dropdown)

**HTML 元素类型**: `<select>` 或自定义下拉组件。

**选项结构**:

```html
<option value="" disabled selected>请选择列…</option>
<option value="0">标题 (文本)</option>
<option value="1">图片 (图片)</option>
<option value="2">描述 (文本)</option>
<!-- 已映射的列不出现在列表中 -->
```

**选项过滤规则**:
1. 从 `fields` 列表中获取所有列。
2. 排除当前所有 `entries` 中已映射的列（通过 `isColumnMapped` 检查），但保留**当前行**已选中的列（因为编辑自己映射行时不应把自己排除掉）。
3. **当当前行已选择源列类型后**，目标图层下拉框做联动过滤（见 4.2.3）。

**占位符显示**:
- 未选择时: "请选择列…"（灰色 italic，placeholder 样式）。
- 已选择时: 显示 `{columnName} ({columnType})`，如 "标题 (文本)"。

**样式**:
- 宽度: 120~140px。
- 字体大小: 11px（Figma 插件 UI 典型字号）。
- 每个 `<option>` 除了列名外，需附加类型后缀 `(文本)` 或 `(图片)`。
- 文本列和图片列可分别以不同颜色小圆点或标签区分（可选增强）。
- 当可用列为 0 时（所有列均已映射），下拉框禁用，显示 "所有列已映射"。

#### 4.2.2 方向指示器

- 固定显示 `→` 箭头符号。
- 字体颜色: 中性灰色 `#8C8C8C`。
- 不可交互，纯视觉装饰。
- 宽度: 20px，居中。

#### 4.2.3 目标图层选择器 (Target Layer Dropdown)

**HTML 元素类型**: `<select>` 或自定义下拉组件。

**选项结构**:

```html
<option value="" disabled selected>请选择图层…</option>
<option value="layer-id-1">Frame1/Grp/Title (文本)</option>
<option value="layer-id-2">Frame1/Grp/Photo (图片)</option>
<!-- 已映射的图层不出现在列表中 -->
<!-- 类型不兼容的图层不出现在列表中 -->
```

**选项过滤规则**（两级过滤）:

1. **类型过滤**: 根据**当前行**已选择的源列的 `columnType`，仅显示 `layerType` 匹配的图层。
   - 若源列尚未选择（placeholder 状态），显示**所有**未映射的图层（不分类型），此时用户可先选图层再选列，选择后会做后置类型兼容校验。
   - 若源列已选择文本列 → 目标下拉框仅显示文本层。
   - 若源列已选择图片列 → 目标下拉框仅显示图片层。

2. **唯一性过滤**: 排除当前所有 `entries` 中已映射的图层，但保留**当前行**已选中的图层。

**显示格式**: `{path}/{layerName} ({layerType})` 或 `{path} > {name} (文本/图片)`。

**占位符显示**:
- 未选择时: "请选择图层…"。
- 已选择时: 显示完整路径 + 类型标签，如 "Frame1 > Header Group > Title (文本)"。
- 若无可用选项（没有兼容类型的未映射图层）: 下拉框禁用，显示 "无可用文本层" 或 "无可用图片层"。

**样式**:
- 宽度: 160~200px（路径可能较长，需截断省略）。
- 路径过长时使用 CSS `text-overflow: ellipsis` 截断。
- 字体大小: 11px。

#### 4.2.4 删除按钮 [×]

- 固定显示红色或中性色的 × 图标按钮。
- 点击后**立即删除**该映射行（无需二次确认）。
- 删除行为:
  1. 从 `entries` 数组中移除该条目的 `id` 对应的 entry。
  2. 该条目中的 `columnIndex` 和 `targetLayerId` 重新变为可用状态。
  3. 其他行不受影响（其下拉框选项在下次渲染时更新）。
- 视觉: `width: 24px; height: 24px;` 居中带 ×，hover 态变为红色 `#FF4D4F`。

### 4.3 添加映射按钮

**位置**: 所有映射行下方，汇总信息栏上方。

**按钮文案**: "+ 添加映射"。

**按钮状态**:

| 条件 | 按钮状态 | Tooltip/辅助文本 |
|------|----------|-------------------|
| 有未映射的列 AND 有兼容类型的未映射图层 | **启用** | 无提示 |
| 所有列已映射 | **禁用** | "所有列已建立映射" |
| 所有兼容图层已映射 | **禁用** | "所有兼容图层已建立映射" |
| 无可用字段 AND 无可用图层 | **禁用** | "无可映射的列或图层" |
| 字段列表为空（未上传文件） | **禁用** | "请先上传数据文件" |
| 图层列表为空（未选择模板或无可用图层） | **禁用** | "请先选择模板" |

**禁用判断逻辑**:

```typescript
function canAddMapping(
  fields: FieldDef[],
  layers: TemplateLayer[],
  entries: MappingEntry[]
): boolean {
  if (fields.length === 0 || layers.length === 0) return false;

  const unmappedFields = getUnmappedColumns(fields, entries);
  if (unmappedFields.length === 0) return false;

  // 至少存在一个未映射列和一个兼容的未映射图层
  for (const field of unmappedFields) {
    const unmappedLayers = getUnmappedLayers(layers, entries);
    const compatible = unmappedLayers.filter(
      l => l.layerType === field.type
    );
    if (compatible.length > 0) return true;
  }

  return false;
}
```

**添加行为**:
1. 创建新的 `MappingEntry`，UUID v4 作为 `id`，`columnIndex: -1`，`targetLayerId: ''`。
2. 将新 entry 追加到 `entries` 数组末尾。
3. 新行渲染在列表底部。
4. 自动聚焦到新行的源列选择器（通过 `autofocus` 属性或 JS `.focus()`）。

### 4.4 映射行状态

每个映射行可以处于以下五种状态之一：

| 状态 | 源列选择器 | 目标图层选择器 | 行背景色 | 说明 |
|------|-----------|---------------|---------|------|
| **空 (Empty)** | 占位符 "请选择列…" | 占位符 "请选择图层…" | 默认 | 新添加的行，尚未做任何选择 |
| **半完成 (Partial)** | 已选择有效列 | 占位符 "请选择图层…" | 默认 | 选了源列但还没选目标图层 |
| **半完成 (Partial)** | 占位符 "请选择列…" | 已选择有效图层 | 淡黄色 `#FFFBE6` | 先选了图层再选列，此行尚未完全可用 |
| **完成 (Complete)** | 已选择有效列 | 已选择有效图层，类型兼容 | 淡绿色 `#F6FFED` | 映射有效，可供生成使用 |
| **错误 (Error)** | 任意 | 任意 | 淡红色 `#FFF1F0` | 类型不兼容或数据异常导致重复映射 |

**状态转换规则**:
- 任一选择器值变更 → 重新评估该行状态。
- 当源列和目标图层**都已选择**且**类型兼容**时，该行进入"完成"状态。
- 当源列和目标图层都已选择但**类型不兼容**时，该行进入"错误"状态，并在行内显示错误消息。

### 4.5 汇总信息栏

**位置**: 映射行列表下方，与映射区域底部固定。

**内容**（一行，水平分布）:

```
已映射: X 列  |  未映射: Y 列  |  未映射图层: Z 个
```

**计算逻辑**:
- `X = entries.length`（每个 entry 占用一个列 → X 即为已映射列数）。
- `Y = fields.length - X`（总列数减去已映射列数）。
- `Z = layers.length - entries.length`（总图层数减去已映射图层数）。

**更新时机**: 每次 `entries` 数组变化（增/删/改映射行）时实时更新。

**样式**: 次要信息色 `#8C8C8C`，字体大小 11px。

### 4.6 批量操作

#### 4.6.1 "清除全部映射" 按钮

- **位置**: 汇总信息栏右侧或下方。
- **文案**: "清除全部映射"。
- **样式**: 文字按钮（无边框、无背景），红色文字 `#FF4D4F`，hover 态加下划线。
- **交互流程**:

```
点击 "清除全部映射"
  │
  ├─ entries.length === 0
  │   └─ 无操作（按钮实际上被禁用）
  │
  ├─ entries.length === 1
  │   └─ 直接清除（无需确认）
  │
  └─ entries.length > 1
      └─ 弹出确认对话框
           "确定要清除全部 X 条映射关系吗？此操作不可撤销。"
           [确定清除]  [取消]
            │
            ├─ 点击 [确定清除]
            │   └─ 清空 entries 数组
            │   └─ 重新渲染 UI（所有下拉框选项恢复可用）
            │
            └─ 点击 [取消]
                └─ 对话框关闭，映射保持不变
```

#### 4.6.2 单行删除

- 点击某行的 [×] 按钮。
- 无需确认，立即删除。
- 删除后该行使用的 `columnIndex` 和 `targetLayerId` 在其他行的下拉框中恢复为可选。

### 4.7 模板变更警告横幅

#### 4.7.1 触发条件

- 从 Sandbox 接收 `selection-changed` 消息 → 新选择的 Frame ID ≠ 当前 `templateId`。
- 或从 Sandbox 接收 `template-layers` 消息 → 附带的新 `templateId` ≠ 当前 `templateId`。

#### 4.7.2 横幅样式

```
┌──────────────────────────────────────────────────────────┐
│  ⚠ 模板已变更，所有映射已清除。请重新建立映射。      [×] │
└──────────────────────────────────────────────────────────┘
```

- 背景色: 淡黄色 `#FFFBE6`。
- 左边框: 黄色 `#FAAD14`，宽度 3px。
- 文字颜色: `#595959`。
- 字体大小: 11px。
- 可关闭: 右侧 [×] 按钮。

#### 4.7.3 数据源变更警告（同等逻辑）

```
┌──────────────────────────────────────────────────────────┐
│  ⚠ 数据源已变更，所有映射已清除。请重新建立映射。    [×] │
└──────────────────────────────────────────────────────────┘
```

- 样式与模板变更警告相同。
- 触发条件: 从 Task 02 相关状态接收到新的 `fields` 列表（文件被重新上传或切换）。

#### 4.7.4 横幅生命周期

1. **出现**: 当变更事件触发时渲染。
2. **关闭**: 用户点击 [×] 后消失。
3. **替换**: 如果新的变更事件发生在旧横幅未关闭时，内容替换为新事件的消息（不叠加多个横幅）。
4. **自动消失**: 不自动消失，必须用户手动关闭（确保用户注意到警告）。

## 5. 交互规范

### 5.1 Dropdown 行为

#### 5.1.1 打开方式

- 使用原生 `<select>` 元素或轻量级自定义下拉组件。
- 点击下拉框打开选项列表。
- MVP 阶段不提供搜索/过滤功能（假设列数 < 20，图层数 < 20）。

#### 5.1.2 选项展示

- 每个选项显示:
  - 源列下拉: `{name} ({type})` 如 "标题 (文本)"。
  - 目标图层下拉: `{path} > {name} ({layerType})` 如 "Frame1 > HeadGroup > Title (文本)"。
- 当前行已选择的值在下拉列表中高亮（`<option selected>`）。
- 不建议使用 `disabled` + `selected` 做 placeholder（原生 select 对 placeholder 的支持不一），可将默认 placeholder option 设为 `display: none` 样式。

#### 5.1.3 自动选择策略

- **不自动选择**。即使用户的源列类型只有一个兼容图层，也不自动选择目标图层。
- 理由: 自动选择可能在用户无意间创建映射，违背"手动建立映射"的产品原则。用户必须显式做出选择。

### 5.2 添加映射行为

1. 用户点击 [+ 添加映射]。
2. 新行在列表最底部渲染。
3. 新行的源列选择器自动获得焦点（`focus()`）。
4. 已存在的映射行不发生变化。
5. 若添加按钮处于禁用状态，点击无反应。建议在禁用按钮上显示 tooltip 说明禁用原因。

### 5.3 删除映射行为

1. 用户点击某行的 [×] 按钮。
2. 该行从 DOM 中移除，对应的 entry 从 `entries` 数组中删除。
3. 该行使用的 `columnIndex` 和 `targetLayerId` 在**其他行**的下拉框中恢复为可选（下次渲染时体现）。
4. 汇总信息栏实时更新。
5. "+ 添加映射" 按钮重新评估启用/禁用状态。
6. 无动画过渡（MVP 不做），DOM 直接更新。

### 5.4 源列选择器的选择变更

1. 用户在源列下拉框中选择一个列。
2. 触发 `onColumnChange(rowId, columnIndex)` 事件。
3. 更新对应 entry 的 `columnIndex`、`columnName`、`columnType`。
4. 联动更新目标图层下拉框的选项:
   - 若新选的列类型为 `'text'`，目标图层下拉框仅展示文本层。
   - 若新选的列类型为 `'image'`，目标图层下拉框仅展示图片层。
   - **若之前已选的目标图层与新列类型不兼容**，清空目标图层选择（设为 placeholder 状态），并在行下方显示短暂提示 "目标图层类型不兼容，已自动清除"（3 秒后自动消失）。
5. 重新评估该行的状态（Complete / Partial / Error）。
6. 重新计算汇总信息。

### 5.5 目标图层选择器的选择变更

1. 用户在目标图层下拉框中选择一个图层。
2. 触发 `onLayerChange(rowId, targetLayerId)` 事件。
3. 更新对应 entry 的 `targetLayerId`、`targetLayerName`、`targetLayerPath`。
4. **后置校验**: 若当前行已选择源列，检查源列类型与目标图层类型是否兼容:
   - 兼容 → 该行进入"完成"状态。
   - 不兼容 → 该行进入"错误"状态，显示行内错误提示。
5. 重新计算汇总信息。

映射模块需监听以下消息：

| 消息来源 | 消息内容 | 处理逻辑 |
|---------|---------|---------|

| 用户操作 | 关闭警告横幅 | 对应 warning 标志设为 `false` |

## 9. 边界情况

### 9.1 极限数据量

- **所有列均已映射**: `canAddMapping()` 返回 false，"+ 添加映射" 按钮禁用。每行的源列下拉框显示 "所有列已映射" 且禁用。
- **所有兼容图层均已映射**: 按钮禁用。仍存在未映射列但其类型无对应未映射图层时，`canAddMapping()` 也返回 false。
- **50 列 3 层**: 映射行区域出现垂直滚动条。最多可建立 3 条映射。其余 47 列在汇总中显示 "未映射: 47 列"。
- **仅 1 列 1 层**: 添加 1 条映射后按钮禁用。用户只能建立一条映射。

### 9.2 空数据

- **空列名**: Task 02 为无表头文件生成默认列名（"列 A"、"列 B" 等），映射 UI 直接使用。
- **0 个字段**: availableFields 为空数组。"+ 添加映射" 按钮禁用。不渲染任何映射行。
- **0 个图层**: availableLayers 为空数组。同上。

### 9.3 类型特殊组合

- **CSV 文件 + 仅有图片层的模板**: 文本列 → 图片层的映射不可能。按钮禁用，所有目标图层下拉框在源列选中文本列后无可用选项。
- **纯文本 Excel + 文本&图片混合模板**: 所有列类型为 text。仅文本层出现在目标下拉框中。图片层在列表中但无法被选中。这是正确的行为。
- **6.4 无表头文件**: 映射 UI 中源列下拉显示 "列 A (文本)"、"列 B (文本)" 等默认名称。

### 9.4 异常操作序列

- **快速连续点击添加映射**: 每次点击创建一个新行，追加到列表末尾。状态逐次更新。
- **先选图层后选列再清空图层**: 该行回到"半完成"状态，背景色恢复。
- **在多个行中同时编辑**: 各自独立，互不干扰。

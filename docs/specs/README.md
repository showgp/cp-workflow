## 5. 技术架构概览

```
┌─────────────────────────────────────────────┐
│                 Figma Plugin                 │
│                                              │
│  ┌──────────────┐    postMessage    ┌──────┐ │
│  │   Sandbox    │◄─────────────────►│  UI  │ │
│  │  (code.ts)   │                   │(html)│ │
│  │              │                   │      │ │
│  │ • Template   │                   │• File│ │
│  │ • Cloning    │                   │• Map │ │
│  │ • Fill       │                   │• Prog│ │
│  │ • Layout     │                   │      │ │
│  └──────────────┘                   └──────┘ │
│       Figma API                      Browser │
│       (no DOM)                       (DOM)   │
└─────────────────────────────────────────────┘
```

- **Sandbox 端**: TypeScript → tsc 编译 → code.js
- **UI 端**: TypeScript + CSS + HTML → esbuild 打包 → ui.html
- **共享层**: src/shared/types.ts 定义所有消息和数据接口

## 6. 各任务规格文档概要

### Task 01 — 项目脚手架与架构

- 定义 23 个源文件的目录结构
- 14 个核心 TypeScript 接口定义
- 11 条消息的双向 postMessage 协议
- esbuild + tsc 混用构建方案
- 16 条验收标准

### Task 02 — 数据源管理

- SheetJS 解析 Excel (.xlsx)，含嵌入图片提取
- CSV 多编码支持 (UTF-8/GBK) 与分隔符检测
- 文件上传 UI (拖拽+点击)
- 字段列表 + 数据行数预览
- 14 种错误码覆盖全部失败路径

### Task 03 — 模板选择与图层发现

- Figma selectionchange 监听 + 300ms 防抖
- Frame 类型校验 (no-selection/multiple/not-frame)
- 递归图层扫描，遇嵌套 Frame/Component 停止深入
- 图层路径构建用于同名区分
- 15 个验收测试用例

### Task 04 — 字段映射系统

- 映射 UI：源列选择器 → 目标图层选择器，逐行操作
- 类型过滤：文本列仅可选文本层，图片列仅可选图片层
- 唯一性约束：列不重复、层不重复
- 模板/文件变更时自动清除映射
- 24 个验收测试用例

### Task 05 — 批量生成引擎

- 三阶段流程：预加载字体 → 逐行克隆替换 → 网格布局排列
- 文本替换保持原样式，图片替换保持原尺寸位置
- 自定义 base64 解码器 (沙盒无 atob)
- 实时进度汇报 + 取消机制
- 空值容错：空文本保留原文，图片失败仅 warning

### Task 06 — 集成、预览与反馈

- 单页面滚动式面板布局，5 个功能分区
- 完整数据流定义 (初始化/上传/选择/生成/取消)
- AppState 状态管理 + 状态驱动 UI 渲染
- 进度条 + 结果摘要 + 零映射确认对话框
- 10 个端到端验收场景

## 7. 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| UI 框架 | 原生 JS | MVP 阶段，插件面板简单，避免框架开销 |
| 打包工具 | esbuild | 极快，适合 UI 代码和库打包 |
| Sandbox 编译 | tsc | 保持 Figma 官方推荐的简洁方案 |
| 图层查找 | 路径匹配 | clone 后 ID 会变，路径唯一标识图层 |
| Excel 图片 | base64 传输 | 沙盒无 File API，需 UI 编码后传递 |
| CSS 方案 | 原生 CSS + 变量 | 无需额外依赖，与 Figma 插件风格一致 |
| 布局网格 | 每行4个 Frame | 适配常见画布宽度，后续可配置化 |

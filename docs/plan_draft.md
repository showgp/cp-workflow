# Figma 插件 MVP 需求文档：表格数据批量填充模板生成器（无 AI 版本）

## 1. 产品目标

设计师/运营人员现在需要把 Excel 中的一批商品、人物、卡片、海报、封面等内容，逐条复制粘贴到 Figma 模板中。

他们真正想要的不是“导入 Excel”，而是：把一份表格里的多行内容，按照一个已设计好的 Figma 模板，批量生成一组结构一致、内容不同的设计页，从而减少重复复制、粘贴、对齐、改图、改文字的时间和错误。

简单的用户故事:

```
作为一个需要批量制作设计稿的人，我已经设计好了一个模板。我不想为 50 行数据重复复制粘贴 50 次。我希望生成结果仍然像我手工排版的一样干净。
我想把表格中的每一行内容自动填入一个 Figma 模板，
以便快速生成一组视觉结构一致、内容不同的页面，
而不是手工重复复制文字、粘贴图片、调整位置和检查遗漏。
```

真正的用户价值是:

```
减少重复劳动
减少复制粘贴错误
保持模板样式一致
更快完成批量设计稿
```

## 2. MVP 一句话描述

用户在 Figma 中选择一个模板 Frame，上传一个 Excel / CSV 文件，手动指定“表格哪一列填入模板哪个位置”，然后插件按表格每一行生成一个新的模板副本，并自动填入对应图片和文字。

该 MVP 不包含 AI。所有字段映射都由用户在插件界面中手动选择。先做一个窄场景 MVP，验证用户是否真的有高频痛点。

我最在意的是:

```
不要做成复杂的万能排版工具
不要一开始支持所有 Excel 格式
不要一开始做 AI 自动理解任意模板
先找到最小闭环
```

## 3. 目标用户

MVP 面向需要批量生成同版式设计内容的用户，例如：

| 用户类型    | 典型任务               |
| ------- | ------------------ |
| 电商运营    | 批量生成商品卡片、促销图、SKU 图 |
| 社媒运营    | 批量生成内容封面、帖子图、活动图   |
| 设计师     | 批量套版生成广告素材         |
| 教育/课程运营 | 批量生成课程卡片、讲师介绍图     |
| 产品运营    | 批量生成产品目录、功能介绍卡片    |

---

## 4. 解决的问题

当前人工流程通常是：

1. 复制一次 Figma 模板。
2. 从 Excel 找到对应行。
3. 复制图片到模板图片位置。
4. 复制标题、描述、价格等文字到模板文字位置。
5. 重复几十次甚至几百次。

MVP 要把这个过程变成：

1. 选择模板。
2. 上传表格。
3. 手动建立字段映射。
4. 点击生成。
5. 自动批量生成结果。

可能的用户旅程:

1. 用户准备一份 Excel 表格
2. 用户在 Figma 中准备一个模板
3. 用户告诉插件：哪一列是标题、哪一列是描述、哪一列是图片, 但也可能只有标题和图片而没有描述.
4. 用户告诉插件：模板中哪些位置对应这些字段
5. 插件预览将生成多少页、哪些行有问题
6. 用户点击生成
7. 插件在画布中生成一组页面
8. 用户检查、微调、交付

## 5 领域对象的解释

Source Table
用户上传或粘贴进来的表格。

Row
表格中的一行，对应一个要生成的设计页。

Field
表格中的一列，例如标题、描述、价格、图片、标签。

Template
用户在 Figma 中准备好的空白模板。

Placeholder
模板中等待被替换的内容位置，例如标题位置、图片位置、描述位置。

Mapping
用户指定的“表格列 → 模板位置”的对应关系。

Generated Page
根据某一行数据生成出来的一页 Figma 设计内容。

Batch
一次生成任务，包含多行数据和多个生成结果。

Issue
某一行或某个字段无法正确填充，例如图片识别失败、文本为空、字段未映射。

## MVP 验收样例

```
Given 用户有一张包含 3 行数据的表格
And 表格中有一列名为「标题」
And 用户选择了一个包含标题占位文本的模板
When 用户把「标题」列映射到模板中的标题文本层
And 点击生成
Then 画布中生成 3 个页面(Frame)
And 每个页面的标题分别来自对应行
And 模板原有字体、字号、颜色尽量保持不变
```

或者:

```
Given 用户有一张包含 4 行数据的表格
And 表格中有一列名为「标题」, 另外一列名为「图片」
And 用户选择了一个包含标题占位文本和图片占位的模板
When 用户把「标题」列映射到模板中的标题文本层, 用户将图片映射到模板中的图片层
And 点击生成
Then 画布中生成 4 个页面(Frame)
And 每个页面的标题分别来自对应行, 每个页面的图片分别来自对应行
And 模板原有字体、字号、颜色保持不变
And 如果图片无法读出(由于是 Excel 的缘故), 则仍接受生成的页面中图片位置是空的状态(图片失败不应阻塞整批生成)
```
